# 泰国达人分发系统 · 达人端 UI/交互/体验优化说明

> 本文档供评审与开发对照，与实现约束一致：**只改表现层**，不删、不并、不改菜单、路由、权限与业务逻辑。

## 约束条件（严格遵守）

1. **不删除、不合并、不修改**现有任何菜单、功能、业务逻辑。
2. **顶部余额 = 积分**，逻辑不变。
3. **中泰双语**正常使用；**不增加翻译延迟、不破坏原有多语言机制**（不新增批量翻译轮询；样式用 CSS 解决泰文行高）。
4. 只做：界面排版、布局、样式、交互反馈、视觉分层、状态展示、体验优化。
5. **积分模式 / 撮合模式**为信息架构与文案区分，**共用**「我的操作日志」；**不新增路由**、不改默认跳转。

## 二、顶部导航（布局优化，不改逻辑）

**从左至右固定顺序：**

菜单折叠按钮 → 系统名称 → 用户名 → **积分余额（大字、加粗、主题色）** → **刷新余额（图标按钮）** → 语言切换（中/泰）→ 消息通知铃铛 → 退出登录。

**实现位置：** DashboardShell.tsx 顶栏 + InfluencerLayout.tsx 传入的 headerExtra（余额与刷新仍绑定现有 getInfluencerPoints() / loadBalance）。

**要求：**

- 余额数字与侧栏角标 **同一 state**，实时一致。
- 刷新按钮：**loading 旋转动画**；成功后 **toast**「余额已更新」（轻量，不阻塞翻译）。
- **用户名固定展示 influencer002（若产品要求）**：仅展示层回退或固定文案，**不修改** getStoredUser() 与登录接口。

## 四、积分模式页面（只改展示）

### 4.1 商家端发单 ClientOrdersHallPage

- 标题：**商家端发单（积分模式）**  
- 说明文字：简短直白。  
- 刷新：主题色大按钮。  
- 搜索框 + 任务状态筛选（映射**现有**状态枚举，不发明新值）。  
- 无数据：图标 + 引导文案 + 刷新按钮。  
- 泰文：line-height 充足，不挤压。

### 4.2 积分与收益 PointsPage

- 顶部三卡：**今日积分、总积分、可提现积分**（字段以**现有接口为准**，无字段则不展示该项，不伪造）。  
- 列表：任务名称、时间、积分、状态；状态用颜色区分。

### 4.3 申请提现 WithdrawPage（重点）

- **单页完成**：存款姓名、银行名称、银行账号、提现金额；**不跳转、不分页**（若接口已有分页逻辑则保持原样，不新增分页 UI）。  
- 前端校验：非空、金额数字、长度合理。  
- 提交：loading + 成功/失败 toast。  
- 提现记录：时间、金额、状态、驳回原因（有则展示）。  
- **不改**提现接口与后端逻辑。

## 七、全局通用

1. 中泰双语与自动翻译机制**不增延迟、不卡顿**。  
2. 泰文：行高加大、不压扁、不重叠。  
3. 色系：金、黄、红、紫为主，有层次，避免过度留白。  
4. 响应式：窄屏侧栏汉堡折叠（已有 isCompact 逻辑）。  
5. 操作反馈：**toast** 轻提示，不遮挡主要内容。  
6. 错误提示：**原因 + 建议**。  
7. 按钮、卡片、文字统一规范。
