## 四、核心业务链路

### 4.1 视频分级订单 流程

```
商家发单 (POST /api/client/market-orders)
  │ 选择 SKU、上传产品图、设定积分、选择等级 ABC
  ▼
管理员/员工审核 (PATCH /api/admin/market-orders/:id)
  │ 审核通过后订单在达人任务大厅展示
  ▼
达人接单 (POST /api/influencer/market-orders/:id/claim)
  │
  ▼
达人制作 → 提交作品 (POST /api/influencer/market-orders/:id/publish)
  │ 上传 work_links
  ▼
管理员验收 → 结算积分 (复核对 / 通过 / 驳回)
  │
  ▼
达人获得积分 → 提现
```

**核心表**: `client_market_orders` → `market_order_applications` → `point_accounts` → `point_ledger`

### 4.2 撮合匹配流程

```
商家创建撮合需求 (MatchingCenterPage)
  │ 填写预算、需求描述、上传附件
  ▼
需求进入达人撮合池
  │
  ▼
达人申请撮合 → 商家从申请人中选择达人
  │
  ▼
分配达人 → 进入合作生产阶段
  │ 分配 → 进行中 → 已提交 → 待审核
  ▼
审核通过 → 待发布 → 已发布 → 已交付 → 已完成
  │                    │
  │                    └── 驳回 → 达人修改重提
  └── 链接验收 (通过/驳回回传链接)
```

**核心表**: `client_market_orders` (order_type=1) → `task_applies` → `cooperation_order_states` → `matching_order_details`

### 4.3 达人撮合权限审核

```
达人填写申请 (InfluencerPermissionPage)
  │ TikTok账号、粉丝数、类目、简介
  ▼
管理员/员工审核 (InfluencerPermissionsPage)
  ├── 通过 → influencer_status = "approved" → 达人可接单/发布需求
  ├── 驳回 → influencer_status = "rejected"
  └── 禁用 → influencer_status = "disabled" (强制关闭)
```

**字段**: `users.influencer_status` (unapplied/pending/approved/rejected/disabled)

### 4.4 SKU 管理

```
商家创建SKU → 上传图片 → 发布订单时勾选SKU → 达人查看SKU参考
    │                                              │
    ├── 批量导入 (Excel / ZIP)                       │
    ├── 编辑 (改名/换图)                             │
    └── 批量删除                                     │
```

**核心表**: `client_skus` | **上传**: multer → `/uploads/skus/{clientId}/`

### 4.5 会员与保证金

```
商家充值保证金 → 冻结部分金额 → 发订单时校验
会员等级: Level 0/1/2/3 (基础/高级/旗舰)
会员开通: purchaseClientMember(level, months) → 扣积分
```

**核心表**: `merchant_profiles` → `member_orders` → `deposit_log`

### 4.6 积分体系

```
积分获取: 商家充值 / 系统赠送
积分消耗: 发布订单 / 达人结算 / 违规扣除
流转记录: point_accounts (余额) ↔ point_ledger (每笔流水)
兑换比例: point_rate_thb (config 表) = 1 积分 = 1 泰铢
```

## 七、鉴权与数据交互

### 鉴权流程
```
1. POST /api/auth/login → { accessToken, user }
2. 前端存储: token 存 localStorage + access_token Cookie
3. fetchWithAuth(): 自动 Bearer header + 401 自动 refresh
4. requireAuth 中间件: JWT 验证 → req.user
5. requireRole('admin','employee'): 角色校验 → 403
```

### 前端数据流
```
Page Component (useState/useEffect)
  → API 层 (adminApi / clientApi / matchingApi / influencerApi)
    → fetchWithAuth()
      → Express Router → Controller → DB query()
        → JSON Response → React State → JSX Render
```

### 文件上传链路
```
1. <input type="file"> → FormData
2. multer 中间件 → /home/ubuntu/influencer-app/uploads/{type}/{id}/
3. DB 存相对路径: /uploads/{type}/{id}/{filename}
4. Nginx 直接 serve /uploads/* (alias)
5. 前端 resolvePublicUploadUrl() 转完整 URL
```

## 八、关键业务流程速查

| 业务场景 | API 序列 |
|---------|---------|
| 商家发视频订单 | POST `/api/client/market-orders` → 管理员审核 PATCH → 达人 `GET /api/influencer/market-orders` → 接单 `POST /claim` |
| 达人接单交付 | `POST /publish` (上传链接) → 管理员 `POST /review` → 积分自动结算 |
| 撮合匹配 | `POST /api/client/matching-orders` → 达人申请 → 商家选择 → 分配生产 → 交付验收 |
| 达人权限审核 | 达人 `POST /api/matching/influencer/permission-apply` → 管理员 `POST /review` (approve/reject) |
| SKU 批量导入 | `GET /import-template` → Excel 填写 → `POST /batch-import` → 图片 `POST /batch-images` |
| 商家会员充值 | `POST /api/matching/client/member/purchase` (level, months) → 扣积分 |
| 保证金充值 | `POST /api/matching/client/deposit/topup` (amount) → deposit_log 记录 |
| 达人提现 | `POST /api/influencer/withdrawals` → 管理员 `PATCH /api/admin/withdrawals/:id` 审核 |
| 结算周表 | `POST /api/admin/settlement/generate` → 按周统计 → `PATCH /:id` 标记已付 |
