# 乱码与编码验收清单

本清单仅用于“编码与文案乱码”验收，不涉及任何业务功能变更。

## 一键命令（推荐）

在 `backend` 目录执行：

```bash
npm run check:release-text
```

该命令会依次执行：

1. 全项目编码与乱码扫描（UTF-8 / BOM / 乱码字符 / JSON charset）
2. 前端构建
3. 后端构建

## 分步命令

### 1) 编码与乱码扫描

在 `backend` 目录：

```bash
npm run check:encoding
```

在 `frontend` 目录（等价）：

```bash
npm run check:encoding
```

### 2) 构建验证

在 `frontend` 目录：

```bash
npm run build
```

在 `backend` 目录：

```bash
npm run build
```

## 验收通过标准

- 扫描输出中以下项均为 `0`：
  - `NON_UTF8`
  - `BOM_UTF8`
  - `GARBLED_TEXT`
  - `JSON_CHARSET_MISSING`
- 前后端构建命令均成功
- 页面中中文/泰文显示正常，无 `?`、`�`、方块、异常转义串

## 脚本位置

- 验收脚本：`backend/scripts/encoding_audit.py`

## 说明

- 数据库为 PostgreSQL 时，字符集应为 UTF8（不是 MySQL 的 `utf8mb4` 命名）。
- 本清单只约束编码与文案显示，不改变业务逻辑。
